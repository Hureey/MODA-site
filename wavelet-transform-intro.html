Wavelet Transform

Introduction

Before reading about the wavelet transform for the first time, please see the introduction to the windowed Fourier transform. This will help you to better understand the wavelet transform. The wavelet transform is a linear transform which uses an adaptive window size to improve time resolution at high frequencies, while still using a large enough window to detect low frequencies. It uses a narrow window at high frequencies for good time resolution, and a wide window at low frequencies for good frequency resolution. [pic of comparison of resolutions in WFT and WT] It does this by using a wavelet (a localised waveform) whose form depends on time and frequency, instead of a window with a fixed width. See examples of wavelets available in the Time Frequency Analysis app in figure below. [wavelet examples from TFA, Morlet, Lognorm, Bump, put wavelet d etails in the appendix/advanced options] The mother wavelet is scaled with parameter s and translated with parameter t. The scale parameter s stretches or compresses the wavelet \psi, and corresponds to the frequency. The time parameter t translates the wavelet. A wavelet with scale s and translation t is compared to the signal and the wavelet transform value at (s,t) quantifies how much the wavelet fits to the signal. If the wavelet correlates very well with the signal, the wavelet transform value is high. This is done for all scales and times, and a 3D picture is built up of time, frequency, and wavelet transform amplitude. [show example contour plot] The wavelet transform has a logarithmic frequency resolution, and therefore a logarithmic frequency scale is used on wavelet transform time-frequency plots. The wavelet transform may also be averaged in time to produce something akin to a Fourier transform, but with better frequency resolution at low frequencies. Previous Next
